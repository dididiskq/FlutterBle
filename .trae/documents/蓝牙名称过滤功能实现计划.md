# 蓝牙名称过滤功能实现计划

## 需求分析

在蓝牙设备列表页面添加名称过滤功能：
- 用户可以输入过滤条件（如"123"）
- 只显示蓝牙名称包含输入内容的设备
- 输入为空时显示全部设备
- 人机交互友好
- 不影响原有功能

## 现有代码分析

### 当前结构
- `_devices`: 存储所有扫描到的设备
- `_isScanning`: 扫描状态
- `_startScan()` / `_stopScan()`: 扫描控制
- `ListView.builder`: 显示设备列表

### 需要修改的部分
1. 添加过滤相关的状态变量
2. 添加搜索框UI组件
3. 实现过滤逻辑
4. 修改设备列表显示逻辑

## 实现方案

### 1. 添加状态变量

```dart
// 过滤文本
String _filterText = '';

// 过滤后的设备列表
List<BleDevice> get _filteredDevices {
  if (_filterText.isEmpty) {
    return _devices;
  }
  return _devices.where((device) => 
    device.name.toLowerCase().contains(_filterText.toLowerCase())
  ).toList();
}
```

**设计考虑：**
- 使用 getter 计算过滤后的列表，避免手动同步
- 不区分大小写，提升用户体验
- 空字符串时返回全部设备

### 2. 添加搜索框UI

**位置：** 在扫描状态提示下方，已连接设备区域上方

**UI设计：**
```dart
// 搜索框区域
Container(
  margin: const EdgeInsets.symmetric(horizontal: 16, vertical: 8),
  decoration: BoxDecoration(
    color: const Color(0xFF1A2332),
    borderRadius: BorderRadius.circular(10.0),
    border: Border.all(color: const Color(0xFF3A475E), width: 1),
  ),
  child: TextField(
    controller: _filterController,
    style: const TextStyle(color: Colors.white, fontSize: 16),
    decoration: InputDecoration(
      hintText: '输入蓝牙名称进行过滤...',
      hintStyle: const TextStyle(color: Colors.grey),
      prefixIcon: const Icon(Icons.search, color: Colors.grey),
      suffixIcon: _filterText.isNotEmpty
          ? IconButton(
              icon: const Icon(Icons.clear, color: Colors.grey),
              onPressed: () {
                _filterController.clear();
                setState(() {
                  _filterText = '';
                });
              },
            )
          : null,
      border: InputBorder.none,
      contentPadding: const EdgeInsets.symmetric(horizontal: 16, vertical: 12),
    ),
    onChanged: (value) {
      setState(() {
        _filterText = value.trim();
      });
    },
  ),
),
```

**人机交互设计：**
- 搜索图标前缀，直观提示搜索功能
- 占位符文本提示用户操作
- 清除按钮只在有内容时显示
- 实时过滤，无需点击确认按钮
- 与现有UI风格保持一致（深色主题）

### 4. 修改设备列表显示

**修改前：**
```dart
ListView.builder(

```

**修改后：**
```dart
ListView.builder(
  itemCount: _filteredDevices.length,
  itemBuilder: (context, index) {
    final device = _filteredDevices[index];
    // 需要找到在原列表中的索引，用于连接操作
    final originalIndex = _devices.indexWhere((d) => d.id == device.id);
    // ...
```

**注意事项：**
- 使用 `_filteredDevices` 显示过滤后的列表

## 不影响原有功能的保证

### 1. 数据流保持不变
- `_devices` 列表仍然存储所有扫描到的设备
- 过滤只是显示层面的操作
- 连接/断开操作仍然基于原始列表

### 2. 扫描功能不受影响
- 扫描逻辑完全不变
- 扫描状态显示不变
- 新设备添加到 `_devices` 列表

### 3. 连接功能不受影响
- 连接操作通过设备ID查找设备
- 连接状态监听不变
- 连接结果处理不变

### 4. UI布局保持一致
- 搜索框插入到合适位置
- 不改变现有组件的布局
- 保持深色主题风格

## 边界情况处理

### 1. 扫描时过滤
- 扫描过程中可以输入过滤条件
- 新扫描到的设备会根据过滤条件显示
- 过滤状态在扫描过程中保持

### 2. 设备连接状态
- 过滤不影响已连接设备的显示
- 已连接设备区域始终显示
- 过滤只影响设备列表区域

### 3. 空输入
- 空输入显示全部设备
- 清除按钮自动隐藏
- 设备数量提示自动隐藏

### 4. 特殊字符
- 支持所有字符输入
- 不对特殊字符做限制
- 过滤逻辑正常处理

## 性能考虑

### 1. 过滤性能
- 使用 `where()` 方法过滤，性能良好
- 设备数量通常不多（<100个）
- 实时过滤不会造成卡顿

### 2. 状态更新
- 只在过滤文本改变时更新状态
- 避免不必要的重绘
- 使用 `setState()` 触发UI更新

### 3. 内存使用
- 不创建额外的设备对象
- 使用 getter 计算过滤列表
- 内存占用可控

本实现方案通过以下方式满足需求：

3. **不影响原有功能**：数据流、扫描、连接等功能完全不受影响
4. **代码简洁**：使用 getter 计算过滤列表，避免手动同步
5. **性能良好**：过滤逻辑高效，不会造成性能问题
